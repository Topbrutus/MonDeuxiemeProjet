# REGLES

## Règles d’or
- Ne jamais tout activer d’un coup si la couche précédente n’est pas validée.
- Tout ce qui vit doit laisser une trace simple.
- Aucun secret ne doit être commité dans le dépôt.
- Tout robot existe dans le registre dès le départ, mais l’activation réelle se fait progressivement.
- On commence avec 3 hamsters pilotes.
- Les générateurs sont testés sur petit échantillon avant extension.
- Toute action externe reste bloquée sans validation humaine explicite.

## Les 7 améliorations intégrées
1. Traduire chaque symbole poétique en fonction technique claire.
2. Standardiser les formats de fichiers, d’événements, d’états et de registres.
3. Limiter les hamsters actifs au début.
4. Retarder l’élimination automatique tant que les métriques sont faibles.
5. Séparer strictement compagnie interne et actions externes.
6. Ajouter un tableau de bord minimal et lisible.
7. Inscrire tous les membres dès le départ, puis les activer progressivement.
